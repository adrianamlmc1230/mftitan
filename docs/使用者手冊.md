# 足球量化分析系統 V2 — 使用者手冊

## 1. 系統簡介

本系統是一套足球賠率數據 ETL 與量化回測工具，將 RPA 抓取的原始 Excel 數據，經過自動化處理流程（前處理 → 隊名比對 → 結算計算 → X 值分類 → 輪次匯總 → 跨賽季合併 → 護級/強度/訊號判定），最終產出各聯賽的決策訊號報表。

技術架構：Streamlit Web 介面 + Pandas 資料處理 + SQLite 本地資料庫。

---

## 2. 安裝與啟動

### 2.1 環境需求

- Python 3.10 以上
- Windows / macOS / Linux

### 2.2 安裝步驟

```bash
# 進入專案目錄
cd football-quant-v2

# 安裝依賴套件
pip install -r requirements.txt
```

### 2.3 啟動系統

```bash
cd football-quant-v2
python -m streamlit run app.py
```

啟動後瀏覽器會自動開啟 `http://localhost:8501`。首頁顯示系統狀態摘要（聯賽總數、啟用數、最近 ETL 執行時間）。

## 3. 功能頁面說明

系統共有 7 個功能頁面，從左側選單進入。

### 3.1 聯賽與賽季管理

路徑：側邊欄 → `1_league_management`

此頁面管理所有聯賽和賽季實例。

#### 匯入 Directory.xlsx

1. 展開「從 Directory.xlsx 匯入」區塊
2. 上傳 `Directory.xlsx` 檔案
3. 點擊「執行匯入」
4. 系統自動解析所有洲別（ASI/AFR/AME/EUR）的聯賽、代碼、前/尾隊伍
5. 匯入完成後顯示新增聯賽數、跳過數、隊伍數

> 注意：已存在的聯賽（依代碼判斷）會跳過，不會重複建立。

#### 手動新增聯賽

展開「新增聯賽」表單，填入洲別、代碼、國家、名稱後送出。

#### 賽季實例管理

1. 從下拉選單選擇聯賽
2. 檢視該聯賽下的所有賽季實例
3. 設定賽季角色：`current`（當季）或 `previous`（上季）
4. 新增賽季實例：填入標識、起始年、結束年、階段

> 重要：ETL 執行時會使用 `current` 和 `previous` 角色的賽季來進行跨賽季合併計算。

---

### 3.2 隊伍分組管理

路徑：側邊欄 → `2_team_groups`

管理每個賽季實例下的隊伍分組（Top / Weak / 自訂）。

1. 選擇聯賽 → 選擇賽季
2. 展開分組卡片，可編輯隊伍清單（每行一隊）
3. 點擊「儲存隊伍」更新
4. 可新增自訂分組，設定方向邏輯：
   - `贏>輸=A`（greater）：贏類大於輸類時產生 A 訊號
   - `輸>贏=A`（less）：輸類大於贏類時產生 A 訊號

> 提示：透過 Directory.xlsx 匯入時，系統會自動建立 Top（前）和 Weak（尾）分組。

---

### 3.3 演算法參數設定

路徑：側邊欄 → `3_parameters`

設定 ETL 計算流程中使用的所有參數。

| 參數 | 預設值 | 說明 |
|------|--------|------|
| X 值區間分界點 | `[-0.24, -0.22, -0.15, -0.08, -0.03, 0.07, 0.15, 0.23]` | 8 個分界點將 X 值分為 9 個區間 |
| 五大區間分組 | `[[1], [2,3,4], [5,6], [7,8], [9]]` | 將 9 區間合併為 5 大區間 |
| 輪次區段大小 | `10` | 每個輪次區段包含的輪次數 |
| 最大區段數 | `6` | 最多計算幾個輪次區段 |
| 護級比值門檻 | `1.4` | 護級判定時的贏/輸比值門檻 |
| 強度升級倍數 | `2.0` | 護級=2 時，MAX/MIN 達此倍數則升級為 4 |

點擊「恢復預設值」可重置所有參數。

---

### 3.4 ETL 執行

路徑：側邊欄 → `4_etl_execution`

執行完整的數據處理流程。

#### 操作步驟

1. 輸入數據根目錄路徑（即 `ERData` 資料夾的路徑）
2. 設定執行範圍：
   - 聯賽：全部或勾選特定聯賽
   - 玩法：HDP（讓球）/ OU（大小）
   - 時段：Early（早盤）/ RT（即時）
3. 點擊「▶ 執行 ETL」
4. 進度條顯示處理進度
5. 完成後顯示結果摘要與品質問題

#### 數據目錄說明

系統不限定目錄結構。只要指定一個根目錄，系統會遞迴掃描該目錄下所有 `.xlsx` 檔案，並從檔名自動解析出聯賽、賽季、玩法、時段等資訊。

你可以使用任意的目錄組織方式，例如按洲別分資料夾、按年份分資料夾，或全部放在同一個資料夾內皆可。

#### RPA 檔案命名規則

```
[國家][聯賽名][賽季年份][第N階段][時段][玩法].xlsx
```

範例：
- `中國中超2025第一階段早亞讓.xlsx`
- `英格蘭英超2025-2026第一階段早大小.xlsx`

---

### 3.5 Report 看板

路徑：側邊欄 → `5_report_dashboard`

檢視 ETL 執行後的決策訊號結果。

1. 篩選條件：玩法（HDP/OU）、時段（Early/RT）、ETL 版本
2. 以洲別分頁顯示（AFR / AME / ASI / EUR）
3. 每個聯賽顯示 5 大區間的訊號（Z1~Z5）
4. 展開聯賽可查看詳細數據：上季贏/輸、當季贏/輸、總贏/輸、護級、強度、訊號
5. 可匯出 Excel 報表

#### 訊號說明

| 訊號格式 | 含義 |
|----------|------|
| `A2` | A 方向，強度 2 |
| `B0.5` | B 方向，強度 0.5 |
| `A4` | A 方向，強度 4（升級後） |
| （空白） | 無訊號 |

---

### 3.6 歷史紀錄

路徑：側邊欄 → `6_history`

檢視所有 ETL 執行的歷史紀錄，包含：
- 執行 ID、開始/完成時間、狀態
- 執行範圍（玩法、時段）
- 展開可查看完整 JSON 資料與品質問題

---

### 3.7 數據驗證

路徑：側邊欄 → `7_validation`

比對舊系統 MST 檔案與新系統 ETL 結果，用於驗證計算正確性。

1. 上傳舊系統的 MST_ 檔案
2. 選擇要比對的 ETL 版本
3. 點擊「執行比對」
4. 檢視比對結果

## 4. 典型操作流程

### 4.1 首次使用

```
1. 啟動系統
2. 聯賽管理 → 匯入 Directory.xlsx
3. 檢查聯賽列表是否正確
4. 隊伍分組 → 確認各聯賽的 Top/Weak 隊伍
5. 參數設定 → 確認或調整演算法參數
6. ETL 執行 → 輸入 ERData 路徑 → 執行
7. Report 看板 → 檢視結果
```

### 4.2 日常更新

```
1. 將新的 RPA 檔案放入 ERData 對應目錄
2. ETL 執行 → 選擇範圍 → 執行
3. Report 看板 → 檢視最新結果
4. 歷史紀錄 → 比較不同版本
```

### 4.3 新賽季切換

```
1. 聯賽管理 → 為需要的聯賽新增新賽季實例
2. 將舊的 current 賽季改為 previous
3. 將新賽季設為 current
4. 隊伍分組 → 更新新賽季的隊伍名單
5. 重新執行 ETL
```

### 5.2 資料庫表格

| 表格 | 說明 |
|------|------|
| `leagues` | 聯賽（洲別、代碼、國家、名稱） |
| `season_instances` | 賽季實例（標識、年份、角色） |
| `team_groups` | 隊伍分組（名稱、方向邏輯） |
| `teams` | 隊伍名稱 |
| `algo_params` | 演算法參數（key-value） |
| `etl_runs` | ETL 執行紀錄 |
| `computation_results` | 計算結果（區間統計、輪次數據） |
| `decision_results` | 決策結果（五大區間、護級、強度、訊號） |
| `quality_issues` | 品質問題紀錄 |

### 5.3 ETL 處理流程

```
RPA .xlsx 檔案
    │
    ▼
前處理（簡繁轉換、方括號清除、數字清除）
    │
    ▼
讀取 → 解析為 MatchRecord 列表
    │
    ▼
隊名比對 → 紀錄拆分至各 TeamGroup
    │
    ▼
結算計算（贏/贏半/輸/輸半/走水 → 數值）
    │
    ▼
X 值分類（9 個區間）
    │
    ▼
輪次區段匯總（每 10 輪一段，最多 6 段）
    │
    ▼
賽季匯總（current + previous 合併）
    │
    ▼
五大區間合併（9 → 5）
    │
    ▼
護級判定（0~3）→ 強度升級（2→4）→ 訊號產生
    │
    ▼
品質檢查 → 結果存入 SQLite
```

## 6. 常見問題

### Q: 匯入 Directory.xlsx 後看不到隊伍？
A: 匯入後請到「隊伍分組管理」頁面，選擇對應的聯賽和賽季，即可看到 Top/Weak 分組及其隊伍。

### Q: ETL 執行時找不到檔案？
A: 請確認數據根目錄路徑正確，且目錄結構符合 `讓球/年份/E_洲別/` 或 `大小/年份/R_洲別/` 的格式。

### Q: 如何重新執行 ETL？
A: 直接再次執行即可，每次執行會建立新的 `etl_run` 紀錄，不會覆蓋舊結果。可在 Report 看板中切換版本比較。

### Q: 資料庫檔案在哪裡？
A: 位於 `football-quant-v2/db/quant.db`，可使用任何 SQLite 工具（如 DB Browser for SQLite）直接查看。

### Q: 如何備份資料？
A: 複製 `db/quant.db` 檔案即可完整備份所有設定和計算結果。
